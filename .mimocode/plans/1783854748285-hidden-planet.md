# SRAMS Enterprise Transformation Plan

> **Objective:** Transform existing SRAMS into a production-grade, multi-portal enterprise education platform with premium SaaS-quality UI, complete RBAC, real-time features, and AI capabilities.
> **Stack:** React 19 + Vite + Tailwind CSS + Zustand + TanStack Table + React Query + Chart.js | Express 5 + PostgreSQL + Socket.io + Cloudinary

---

## Current State Assessment

**What exists and works:**
- Backend: 29 controllers, 30 routes, comprehensive JWT auth (2FA, OTP, PIN login, session management, audit logs)
- Database: 30+ PostgreSQL tables with migrations, indexes, foreign keys
- RBAC: 15 roles with ~50 permissions defined in `permissions.js`
- Frontend: 3 portal layouts (Admin/Teacher/Student), 31 pages, route protection, AuthContext
- Email: 13 Handlebars templates, Nodemailer service
- Cron: Database backup, attendance sync

**What's missing or broken:**
- Parent portal (no pages, no routes)
- Permission middleware defined but NOT wired to routes (only `authenticate` is used)
- 8 admin pages are read-only stubs (Users, Attendance, Assessments, Marks, ReportCards, Promotions, Notifications, GradingSystem)
- Settings.jsx has bugs (`fetchSubjects` undefined, `LoaderPinwheelIcon` missing import)
- No Zustand stores (declared in package.json, not used)
- No React Query (not installed)
- No TanStack Table (not installed)
- No Zod validation (not installed)
- No React Hook Form (not installed)
- No Chart.js integration (not installed)
- No Socket.io (not installed)
- No Cloudinary (not installed)
- No Tiptap rich text editor (not installed)
- No dark mode
- No Swagger/OpenAPI docs
- No tests
- No seed data
- No deployment config (PM2, GitHub Actions)

### 3.3 Wire Permission Middleware to All Routes

**Critical gap:** `permissions.js` exists but routes only use `authenticate` + `authorize()` (role-based). Need to add `requirePermission()` to every route for fine-grained access control.

**Modify:** All 30 route files in `backend/routes/router/`

Pattern for each route:
```javascript
router.get('/',      authenticate, requirePermission('students:read'), getAll);
router.post('/',     authenticate, requirePermission('students:crud'), create);
router.put('/:id',   authenticate, requirePermission('students:crud'), update);
router.delete('/:id', authenticate, requirePermission('students:crud'), remove);
```

### 3.4 API Response Standardization

**Create:** `backend/utils/apiResponse.js`

Standard response format:
```javascript
{ success: true, data: {...}, message: "...", meta: { page, limit, total, totalPages } }
{ success: false, error: "...", code: "VALIDATION_ERROR" }
```
