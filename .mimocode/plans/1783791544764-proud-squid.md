## Phase 3: Authentication & RBAC

### Backend Changes

**auth.controller.js â€” Student PIN Login:**
```
POST /api/v1/auth/students/login-pin
Body: { admission_no, pin }
Logic: Find student â†’ Check lockout â†’ Verify PIN â†’ Reset attempts â†’ Find/create user â†’ JWT
```

**authorization.js â€” Resource-Level Checks:**
- `canAccessStudent` â€” Admin: all; Teacher: assigned classes; Student: own data only
- JWT payload: `{ id, role, studentId }` for students

### RBAC Mapping
| Resource | Admin | Teacher | Student |
|----------|-------|---------|---------|
| `/students` | CRUD all | Read assigned | Read self |
| `/teachers` | CRUD all | Read self | - |
| `/attendance` | CRUD all | Mark own classes | Read own |
| `/marks` | CRUD all | Enter own subjects | Read own |
| `/assessments` | CRUD all | CRUD own subjects | Read own |
| `/report-cards` | CRUD + generate | Read assigned | Read own |
| `/grading-system` | CRUD all | Read | Read |
| `/notifications` | Send to all | Read own | Read own |
| `/trades`, `/classes`, `/subjects` | CRUD | Read assigned | Read assigned |
| `/users` | CRUD all | - | - |
| `/backups` | Full | - | - |
